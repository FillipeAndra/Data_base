
- os dados são modelados com base nos requisitos do cliente e/ou produto

- os requisitos podem ser funcionais e não funcionais

## Requisitos Funcionais

- adicionam uma funcionalidade ao sistema podendo ser necessários ou não para o sistema funcionar

## Requisitos Não Funcionais

- São as necessidade para que os requisitos funcionais funcionem

## Modelagem do Banco

- A construção do banco de dados entra no processo de requisitos não funcionais, onde para que ele seja modelado para atender as necessidades de armazenamento de dados necessárias para o sistema em análise


- A modelagem é composta por 3 fases

### Modelagem conceitual

- Na nomenclatura a tabela pode ser chamada de entidade

- Se baseia na  elaboração de um documento enumerando o que vai ser necessário para cada tabela e quais os nomes das tabelas vão fazer parte do banco e as relação entre as tabelas. 

- Esse documento pode ser um diagrama ou apenas uma documentação em texto mesmo

- geralmente é feita pelo AD (administrador de dados) que é responsável por todos os bancos de dados da empresa
### Modelagem lógica

- É o processo de catalogar quais vão ser os tipos de dados de cada coluna

- informa quem é a primary key e a foreing key de cada tabela

- geralmente é feita pelo AD (administrador de dados) que é responsável por todos os bancos de dados da empresa
### Modelagem física

- processo de escrita do script em SQL

- geralmente feito pelo DBA (Administrador do banco de dados) e/ou AD
